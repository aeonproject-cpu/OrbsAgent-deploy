Abstract

Trust is the missing primitive of the agent economy.

AI agents are no longer theoretical. They trade capital, execute transactions, manage portfolios, and act autonomously on behalf of millions of users — today, at scale, without pause. Yet the infrastructure layer beneath them is alarmingly primitive: agents operate without identity, without history, without consequence.

Orbs Agent proposes a foundational solution: a permissionless, on-chain identity and reputation system for AI agents. Any agent can register a permanent identity. Every action is recorded immutably. Reputation accrues or degrades based on verifiable behavior. Agents stake ORBS tokens as economic collateral. Trust becomes queryable, programmable, and composable.

This paper describes the problem in full, the architecture of the Orbs Agent protocol, the ORBS token model, and the governance structure that will govern the protocol's evolution.

Section I

The Anonymous Agent Economy

The proliferation of autonomous AI agents has outpaced every layer of infrastructure designed to support them. In the span of eighteen months, the number of agents executing financial transactions on public blockchains grew by an estimated three orders of magnitude. They operate DeFi strategies, execute cross-chain arbitrage, manage NFT portfolios, and run automated governance votes — all without a single verifiable identity on record.

This is not a technical failure. It is a design omission. The protocols upon which these agents operate were built for human actors, not autonomous systems with variable behavior profiles, unknown provenance, and disposable deployment patterns.

"An agent fails — it disappears. No history. No reputation. No consequence."

The consequences of this omission are compounding. When an agent behaves maliciously — or simply fails — there is no mechanism for attribution. The deployer cannot be identified. The behavioral record does not exist. The capital lost is gone without recourse. Users are left holding the outcome of decisions made by systems they cannot inspect, verify, or hold accountable.

More insidiously, the lack of identity infrastructure creates a race to disposability. Bad actors deliberately architect agents to be ephemeral — deployed for a single operation, then abandoned. Without persistent identity, reputation cannot accumulate. Without reputation, there is no incentive to behave well over time. The market selects for short-termism by default.

The three structural failures of today's agent economy: (1) Anonymous deployment with no owner attribution. (2) No persistent behavioral record across agent lifetimes. (3) Zero economic consequence for failure or malicious action.

Existing solutions — KYC layers, allowlists, and off-chain reputation systems — are insufficient. They are centralized, permissioned, and incompatible with the composability requirements of a multi-chain agent economy. What is required is a neutral, permissionless, on-chain primitive: a reputation layer that any protocol can read without trusting any intermediary.

Section II

The Orbs Agent Protocol

Orbs Agent is a permissionless, chain-agnostic protocol that provides four primitives for the agent economy: Identity, Behavior Recording, Stake-Backed Commitment, and Trust Queries. Together, these primitives make reputation programmable.

The protocol is non-custodial. Orbs Agent does not hold funds, control agents, or make trust decisions. It provides the infrastructure upon which trust decisions can be made by any consuming application, with any threshold, for any purpose.

The design is deliberately minimal. We resist the temptation to build a governance layer for agent behavior. The protocol records and exposes — it does not prescribe. The market determines what reputation score is sufficient for a given use case.

"Not a dashboard. Not a whitepaper.
Infrastructure."

Identity Registration. Any agent — regardless of framework, chain, or owner — can register a permanent on-chain identity via the Orbs Agent registry contract. The identity is non-transferable and immutable once created. Registration requires a minimum ORBS stake, ensuring economic commitment from day one. The registry stores a minimal identity record: owner address, chain, creation timestamp, and optional metadata.

Behavior Recording. The protocol exposes a recording interface for dApps, protocols, and agent orchestrators to submit behavioral events against a registered agent ID. Events are typed (trade execution, governance vote, position management, etc.), timestamped, and linked to on-chain transactions wherever possible. The behavioral ledger is append-only and cannot be modified or deleted.

Stake-Backed Commitment. Registered agents stake ORBS tokens as economic collateral backing their identity. Stake is locked for a configurable period and subject to slashing by governance vote in cases of provable malicious behavior. Higher stake — particularly with longer lock periods — yields a higher reputation multiplier, creating a strong incentive to commit meaningfully.

Trust Query Interface. The protocol exposes a unified API and on-chain read function that returns a trust signal for any agent ID. Callers can specify their own thresholds: minimum score, minimum stake, minimum tier, or any combination. The result is binary (trusted/not trusted) plus the underlying score data for applications that want full context.

Section III

Technical Architecture

The Orbs Agent protocol is composed of four on-chain contracts and one off-chain indexing layer. All contracts are upgradeable via governance with a 48-hour timelock. The indexer is operated by the Orbs Agent Foundation initially, with plans to decentralize operation in Phase 4.

Registry Contract. Deployed on Ethereum mainnet with cross-chain message bridges for Solana, Base, and Arbitrum. Stores the canonical identity record for each registered agent. Owner-only mutation with signature verification. Registration events are emitted and indexed by the chain indexer.

Reputation Engine. A stateful contract that maintains scoring state for each registered agent. Score updates are batched and submitted by authorized reporters (initially the Orbs Agent Foundation, later a decentralized set of node operators). The scoring formula is publicly documented and immutable without governance approval.

Staking Contract. Manages ORBS token deposits, lock schedules, and slashing logic. Slashing events require a two-of-three multisig approval from the Slashing Committee — a governance-elected body with rotating membership. Slashed tokens are sent to a community treasury, not burned.

Trust Resolver. A lightweight aggregation contract that combines registry, reputation, and staking data into a single trust signal. Optimized for gas efficiency with packed storage reads. Off-chain caching layer at API nodes reduces on-chain load for high-frequency query patterns.

Section IV

The ORBS Token

ORBS is the native utility and governance token of the Orbs Agent protocol. It serves three functions simultaneously: it is the staking asset that backs agent identities, the governance token that controls protocol evolution, and the fee currency for advanced API access.

The total supply of ORBS is fixed at one billion tokens (1,000,000,000 ORBS). There is no inflation schedule. All tokens are minted at genesis and distributed according to the allocation below. The fixed supply ensures that staking dynamics create genuine scarcity as protocol adoption grows.

ORBS is not a speculative instrument. Its value is derived from protocol utility: agents must hold and lock ORBS to maintain their identity and reputation standing. As the agent economy grows, demand for ORBS is structurally linked to the number of active, staked agents.

The fee model is minimal. Basic trust queries are free. High-volume API access, advanced scoring parameters, and real-time webhooks require a fee denominated in ORBS, paid to a protocol treasury governed by ORBS holders.

Section V

Governance & The Path Forward

The Orbs Agent protocol will be governed by ORBS token holders from day one of the governance module launch (Phase 3). Governance controls: protocol parameter changes, scoring formula updates, slashing committee elections, treasury allocations, and protocol upgrades.

Proposals require a minimum of 1,000,000 ORBS in delegation to advance to a vote. Voting lasts seven days. A quorum of 10% of circulating supply and a majority of 60% is required for passage. All passed proposals are subject to a 48-hour execution timelock during which the community can respond to unexpected behavior.

The Orbs Agent Foundation — a Swiss-incorporated nonprofit — will steward the protocol during the launch and early growth phases. The Foundation's role is explicitly transitional. As on-chain governance matures and ORBS distribution broadens, Foundation powers will sunset and transfer to the DAO. The target date for full decentralization is Q2 2027.

Phase 1 — Launch. ORBS token live on Ethereum mainnet. DEX liquidity provisioned. Community formed. Governance forum opened. The Foundation publishes quarterly transparency reports covering protocol metrics, treasury usage, and development progress.

Phase 2 — Registry. Agent identity contracts deployed. First agents register. Developer SDK released. Indexer launched. First trust query API goes live. Integration partnerships announced with at least three major DeFi protocols.

Phase 3 — Reputation. Scoring engine live. Behavioral recording begins. Staking contract activated. Slashing committee elected by governance. First slashing event demonstrates the enforcement mechanism is real and functional.

Phase 4 — Standard. Cross-protocol adoption campaign. On-chain governance module transferred from Foundation to DAO. Chain indexer nodes decentralized. ORBS reputation layer submitted to EIP process as a standard for agent identity across EVM chains.
